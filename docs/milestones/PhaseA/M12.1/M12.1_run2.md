## Step 4 â€” Failure Analysis

### Failure 1: Test Failures (175 failures)

**Classification:** Correctness bug (architectural limitation)

**Root Cause:**
The `model_validator(mode="before")` approach **does not intercept nested model dicts** created by Pydantic during parent model validation. When Pydantic validates a parent model containing a list of child models, it:
1. Converts child model instances to dicts using `model_dump()` (field names, snake_case)
2. Passes these dicts to nested validation
3. **Bypasses the `model_validator`** on the child model class
4. Validation fails because dicts have snake_case keys but validation expects camelCase aliases

**Evidence:**
- Direct dict validation works: `PieceFeatures.model_validate({"slot_id": 0, ...})` âś…
- Nested model validation fails: `PerPieceFeaturesV1(pieces=[PieceFeatures(...), ...])` âťŚ
- Local testing confirms validator works for explicit calls but not for nested validation

**In Scope:** âś… Yes â€” this is the core M12.1 objective

**Blocking:** âś… Yes â€” blocks M12.1 completion

**Resolution Path:**
The `model_validator` approach is **architecturally sound** but **does not work for nested model validation** in Pydantic v2.0.3. We need to investigate:
1. Whether Pydantic v2.0.4+ fixes this behavior
2. Whether a different interception point is needed (e.g., `field_validator`, `model_serializer`)
3. Whether we need to override `model_dump()` to use aliases when serializing for nested validation

**Possible Solutions:**
1. **Field Validator Approach:** Use `field_validator` on list fields to normalize dicts before they're passed to nested validation
2. **Model Dump Override:** Override `model_dump()` to use aliases when serializing for nested validation
3. **Model Serializer:** Use `model_serializer` to control how models are serialized to dicts
4. **Pydantic Version:** Check if Pydantic v2.0.4+ fixes this behavior

**Recommendation:** Investigate the `field_validator` approach first, as it may be the most direct interception point for list fields containing nested models.
